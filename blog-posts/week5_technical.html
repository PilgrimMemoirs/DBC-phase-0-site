When, Why & How to Classes in Ruby

 

Sometimes in ruby you'll have multiple objects that require the same actions(methods), that's where using a class comes handy. Especially when those similiar objects start reaching large large quantities, you'll want a way to automate thier actions. In the example below I created a class that takes a farm animal and the quanity of that animal as it's initial input to find out how much food and housing you'll need for all your animals.

Example

class FarmAnimal #Define the Class. Class names use CamelCase

def initialize(quantity) #An initialize method. Will expect 1 argument, in this case the quantity of that particular farm animal

@quanity=quantity #The argument is turned into an instance variable so it can be used throughout the class. An @ before the name of the variable makes it an instance variable.

end

def needs #This will be the method you call to find out what you need for each breed of animal. It calls instance varibles set below.

p @stays

p @eats

end

def eats(food, amount_per_day) #A method to calculate how much food you'll need per day for each breed. Gives output string sentence saying exactly what you need.

amount_of = @quanity * amount_per_day

@eats ="You need #{amount_of} #{food} per day."

end

def stays(housing, acre) #A method to calculate how many acres of the kind of housing you'll need for each breed. Gives output string sentence saying exactly what you need.

amount_of = @quanity * acre

@stays ="You need #{amount_of} acres of #{housing}."

end

end

chicken = FarmAnimal.new(10)

chicken.eats("cups of pellets", 0.5)

chicken.stays("range", 0.25)

cow = FarmAnimal.new(5)

cow.eats("bales of hay",2.5)

cow.stays("pasture",1)

pig = FarmAnimal.new(3)

pig.eats("buckets of slop", 2)

pig.stays("pig pen", 0.5)

p pig.needs

p chicken.needs

p cow.needs
